Local GitOps Plan
k3s + GitHub + ArgoCD

A complete plan for setting up a local GitOps cluster on a separate machine in the same network, portable to EKS later.

PREREQS

Required Tools

Cluster machine
k3s
Linux host on local network
Main machine
kubectl + helm
Remote cluster access via kubeconfig
GitOps
ArgoCD + argocd CLI
Sync engine
Version control
GitHub repo
Single source of truth
ℹ️ k3s replaces kind/minikube. It runs as a real Linux service on the cluster machine, survives reboots, and exposes the cluster over LAN — no Docker-in-Docker limitations.
STEP 1

Create the Repo Structure

github.com/you/gitops-local

Create a new GitHub repo and scaffold the following directory layout. This is the single source of truth ArgoCD will watch.

gitops-local/
├── bootstrap/
│   └── root-app.yaml      # App of Apps — points to /platform and /apps
├── platform/
│   ├── ingress-nginx/
│   │   └── application.yaml  # ArgoCD Application for ingress-nginx
│   └── sealed-secrets/
│      └── application.yaml  # ArgoCD Application for sealed-secrets
└── apps/
    ├── app-a/
    │   ├── application.yaml  # ArgoCD Application manifest
    │   └── manifests/
    │      ├── deployment.yaml
    │      ├── service.yaml
    │      └── ingress.yaml
    └── app-b/
        ├── application.yaml
        └── manifests/
           ├── deployment.yaml
           ├── service.yaml
           └── ingress.yaml
Local equivalents (vs EKS)
  • NGINX ingress instead of AWS LB Controller
  • Sealed Secrets instead of ESO + Secrets Manager
  • k3s local-path instead of EBS CSI driver
Sample apps
  • app-a → nginx serving "Hello from App A"
  • app-b → nginx serving "Hello from App B"
  • Both exposed via Ingress on separate hostnames
STEP 2

Install k3s on the Cluster Machine

run on cluster machine

SSH into the cluster machine and run the following. Traefik is disabled since we'll use NGINX ingress instead.

cluster machine — bash
# Install k3s, disabling the built-in Traefik ingress controller
curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="--disable traefik" sh -

# Verify the node is ready
kubectl get nodes

# Print the kubeconfig (you'll need this on your main machine)
cat /etc/rancher/k3s/k3s.yaml
⚠️ Note the cluster machine's local IP address (e.g. 192.168.1.x). You'll need it in the next step to replace 127.0.0.1 in the kubeconfig.
STEP 3

Configure Remote kubectl Access

run on main machine

Copy the kubeconfig from the cluster machine to your main machine and update the server address.

main machine — bash
# Copy kubeconfig from cluster machine (replace with actual IP)
scp user@CLUSTER_IP:/etc/rancher/k3s/k3s.yaml ~/.kube/k3s-config

# Replace 127.0.0.1 with the cluster machine's LAN IP
sed -i 's/127.0.0.1/CLUSTER_IP/g' ~/.kube/k3s-config

# Point kubectl at the new config
export KUBECONFIG=~/.kube/k3s-config

# Verify remote access
kubectl get nodes
💡 Add the export KUBECONFIG line to your ~/.bashrc or ~/.zshrc so it persists across terminal sessions.
STEP 4

Bootstrap ArgoCD

run on main machine

Install ArgoCD via Helm into its own namespace. This is the only manual install — from here ArgoCD manages everything else.

main machine — bash
# Add the ArgoCD Helm repo
helm repo add argo https://argoproj.github.io/argo-helm
helm repo update

# Install ArgoCD
kubectl create namespace argocd
helm install argocd argo/argo-cd -n argocd

# Wait for pods to be ready
kubectl wait --for=condition=ready pod -l app.kubernetes.io/name=argocd-server -n argocd --timeout=120s

# Port-forward the ArgoCD UI (use cluster machine IP)
kubectl port-forward svc/argocd-server -n argocd --address 0.0.0.0 8080:443

# Get the initial admin password
argocd admin initial-password -n argocd
ℹ️ ArgoCD UI will be accessible at https://CLUSTER_IP:8080 from any machine on the network. Accept the self-signed cert warning.
STEP 5

Connect ArgoCD to GitHub

run on main machine

Register your GitOps repo with ArgoCD using a GitHub deploy key or personal access token.

main machine — bash
# Login to ArgoCD CLI
argocd login CLUSTER_IP:8080 --insecure

# Register the GitOps repo (HTTPS with token)
argocd repo add https://github.com/YOU/gitops-local \
  --username YOUR_GITHUB_USER \
  --password YOUR_GITHUB_PAT

# Verify the repo is connected
argocd repo list
🔑 For a more production-like setup, use an SSH deploy key instead of a PAT: argocd repo add git@github.com:YOU/gitops-local --ssh-private-key-path ~/.ssh/id_ed25519
STEP 6

Apply the Root App of Apps

triggers full sync

Apply the single root Application manifest. ArgoCD will discover all platform tools and apps from this one entry point and sync them automatically.

main machine — bash
kubectl apply -f bootstrap/root-app.yaml -n argocd

# Watch ArgoCD sync all discovered applications
argocd app list

# Watch sync status live
argocd app wait --all
bootstrap/root-app.yaml — example
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: root
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/YOU/gitops-local
    targetRevision: HEAD
    path: .
    directory:
      recurse: true
  destination:
    server: https://kubernetes.default.svc
    namespace: argocd
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
STEP 7

Configure Local DNS

reach apps from browser

Point the app hostnames to the cluster machine's LAN IP so you can reach them from your main machine's browser.

main machine — /etc/hosts
# Add these lines (replace with actual cluster machine IP)
192.168.1.x   app-a.local
192.168.1.x   app-b.local
💡 If this cluster will stick around long-term, consider running dnsmasq or Pi-hole on your network to handle *.local as a wildcard — avoids editing /etc/hosts for every new app.

EKS MIGRATION

What Changes When Moving to EKS

The gitops-repo structure stays identical. Only the platform layer swaps out cloud-specific tools.

Concern Local (k3s) AWS (EKS)
Cluster provisioning k3s install script Terraform + eksctl
Ingress controller ingress-nginx AWS LB Controller
Secrets management Sealed Secrets ESO + Secrets Manager
Storage k3s local-path EBS CSI driver
DNS /etc/hosts or dnsmasq ExternalDNS + Route53
Container registry Docker Hub / GHCR ECR
ArgoCD itself Identical — same Helm chart, same config
App manifests Identical — no changes needed
